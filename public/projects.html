Video Grabber
With the help of React (along with webpack and babel) this utility will find, retrieve and display video data based on search terms and a few options. Jobs are created in the NodeJS backend using Bull, where each job performs a web scraping task with Puppeteer, which runs a headless browser which will search google for the term, visit each url of the search and retrieve the video along with a dialogue transcript if available. Search requests can be done simultaneously, can be cancelled and each video can be removed from the results list once no longer needed.

Event Dialogue Search
This website uses transcriptions of dozens of events from a spiritual teacher named Bashar. Transcriptions of the event dialogue were generated by AWS Transcribe and then pre-formatted through custom scripts. Dialogue snippets can be searched for with keywords, and recordings of the dialogue can be listened to alongside realtime transcript highlighting. The web app also allows a second search option that will web scrape their website to find events where a certain question containing the specified keyword were asked. The app is created with NodeJS on the backend and React on the front. Event audio clips are stored in AWS S3.

Snappir
Used as my capstone project for my Associate's Degree, Snappir is meant to act as a social platform for landscape and cityscape photos. You can sign up and upload your own photos, comment on other posts, message and follow other users and show support by liking and favoriting posts. The site features infinite scrolling and post filtering, as well as Google Maps integration. This was my first implementation of some of these concepts and an app of this size, so of course many parts of the app aren't quite optimal. Written with NodeJS and MongoDB as the database.
